Androidmanifest 修改实践

发表于 2022-03-13 分类于 Android逆向实践 阅读次数：

根据之前的 Androidmanifest 混淆样本，总结出其中的静态混淆方案并使用 Android Studio 编写 demo 进行实践。

本次实践使用 010Editor 作为 Androidmanifest 十六进制修改工具，使用 apktool 对修改过的 apk 进行反编译，之后再将 apk 重签名运行，检测 apk 功能是否受到影响。

  1. 修改文件开头的 magic number，将其修改为 00080000h，修改后 apktool 反编译失败，apk 正常运行未受影响。
  2. 修改 string count，使其值大于实际值，修改后 apktool 反编译失败，010Editor 模板将 string pool 中的值作为 offsets 读入。该修改不会影响 apk 正常运行。
  3. 修改命名空间前缀，该值在正常 apk 中通常为 android，将其修改为乱码字符，并修改开头表示字符串长度的字段。apktool 正常反编译，但是所有的 android 标识都变为我所设置的乱码字符串，如果将其设置为一个长度足够的乱码字符串，将很影响静态分析。该修改不会影响 apk 正常运行。
