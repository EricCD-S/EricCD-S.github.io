Androidmanifest 修复实践

发表于 2022-04-01 更新于 2022-04-05 分类于 Android逆向实践 阅读次数：

记录一次 Androidmanifest.xml 的修复实践过程。

工具准备

本次修复使用 apktool 作为 APK 反编译工具，并使用 010Editor 对 Androidmanifest.xml 文件进行十六进制修改。010Editor 中的官方 Androidmanifest 模板对于修改实践有很大的帮助。

修复过程

  1. 使用 jadx 打开样本 apk，AndroidManifest 能够正常打开，但是 start chunk 处出现错误格式，发现在命名空间赋值的语句前出现了多余的语句。
  2. 使用 010Editor 打开 xml 文件，首先发现其在 start namespace chunk head 中的 chunk type 字段出错，文件中的值为 00011400，但正确的值应该为 00011000。
  3. 继续检查 start namespace chunk 的内容，发现有两处异常点。首先，Size 字段的值通常为 24，此处多出了 4 字节，刚好为 chunk 中一个字段数值的长度。其次，prefix 的值通常为在 string pool 中的偏移量，而该处的值远远超出 string count。检查从 prefix 开始的 12 字节，检查后发现，后面 8 字节的数据分别对应 string pool 中的 android 和 url，因此，考虑删去前面的异常数值，并修改 size 大小。
  4. 使用 jadx 重新打开修改后的样本，发现已经恢复正常
